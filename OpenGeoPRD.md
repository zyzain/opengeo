1. 产品演进概述
基于已搭建的 GEO 开源系统，本次 PRD 旨在完成从“功能堆叠型工具”到“插件化开源框架”的跃迁。保留现有 AI 内容优化、多账号矩阵、多渠道发布等实战能力，通过引入 Protobuf 契约驱动、六边形架构、品牌数字孪生 三大核心改造，解决现有系统扩展性弱、AI 生成缺乏品牌约束、社区贡献门槛高的问题。

2. 功能架构详解（框架化重构版）
2.1 品牌数字孪生中心（新增核心层）
改造重点：在现有“内容管理”之上构建品牌元数据层，作为所有 AI 与诊断的基准
品牌空间建模：支持 VI 规范、语调描述、术语表、受众画像的结构化存储，替代原有扁平标签体系。
知识图谱构建：将非结构化品牌资料转化为 AI 可理解的实体关系，支撑事实校验与引用归因。
多版本快照：支持品牌升级前后对比诊断，历史元数据可回溯。
租户级隔离：现有“多租户”能力增强，强制行级安全过滤，支撑集团客户多品牌独立治理。
2.2 AI 内容优化工厂（增强现有能力）
改造重点：从“通用文案生成”升级为“品牌约束生成”
Prompt 工程框架：所有 AIGC 请求自动注入品牌元数据 + 术语表，确保输出一致性。
合规预审引擎：集成规则引擎 + 轻量 NLP 模型，生成即校验敏感词、版权风险、品牌事实准确性。
多语言自适应：一次生成自动适配 20+ 语言并保留术语，BLEU 评分阈值校验。
流式交互体验：SSE 推送 Token 级内容，前端实时渲染 + 违规片段高亮反馈。
2.3 可插拔诊断引擎（重构监测分析）
改造重点：将硬编码的评分逻辑抽象为插件接口
DiagnosticPlugin 接口：定义标准化的输入（品牌实体+内容+指标）、输出（分数+归因+建议）契约。
内置基础维度：保留现有信源评分、竞品监测作为官方插件，新增语义相关性、引用权威性等《GEO Guide 2025》推荐维度。
归因链路可视化：从指标波动直达底层数据（如某篇负面文章导致情感分下降），关联 Jaeger Trace。
实时仪表盘：SSE 推送指标变化，Web Worker 防主线程阻塞，ECharts 雷达图动态更新。
2.4 标准化发布与调度（重构发布管理）
改造重点：解耦渠道实现，支持社区贡献
ChannelAdapter 接口：统一微信/微博/抖音/小红书等渠道的发布、预览、状态回写契约。
矩阵策略参数化：将“权威背书层/专业认证层/生态渗透层”分组策略配置化，AI 根据历史 ROI 自动推荐发布频次与时段。
异步任务队列：EDA 事件驱动发布任务，支持重试、熔断、状态回调。
发布日历增强：AI 活跃度热力图 + 智能排期建议 + 冲突检测。
2.5 企业级协同基座（增强系统设置）
改造重点：明确开源与企业版边界，提升可观测性
Proto First 契约层：所有 API 由 .proto 定义，CI 自动生成 Go/TS 代码，Breaking Change 阻断合并。
全链路可观测性：Hertz → Kitex → DB/LLM 调用均携带 TraceID，Grafana 预置 QPS/延迟/错误率面板。
RBAC + SSO：核心 RBAC 开源；SSO/LDAP/审计日志作为 Enterprise 插件，Feature Flag 控制。
插件市场：诊断模型、分发渠道、评估算法均通过 registerPlugin() 注册，核心代码零硬编码。
